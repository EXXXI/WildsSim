# WildsSim(気ままに作ったモンハンシミュレータ for MHWilds)

## **TU4向けalpha版の注意点**

20251206.alpha版にて、12/16配信予定のTU4への対応を行いました。

当然ながら**各種仕様は事前情報からの推測が多く含まれており、実機と挙動が違う可能性があります。**

また、TU4配信後に実機に合わせて修正を行うため、場合によっては**alpha版のデータが正式版に正常に引き継げない可能性もあります。**(もちろん可能な限りそうならないようには努力しますが)

alpha版は上記を承知の上でTU4環境のシミュを行いたい方のみご利用ください。

安定動作を求める方はTU4対応の正式版ができるまで、20251115.0.1版をご利用ください。

どちらも[releases](https://github.com/EXXXI/WildsSim/releases)の画面から入手できます。

### TU4向けalpha版の仕様について

- アーティア関連
  - スキル付きのアーティア武器を登録し、検索に利用できます。
  - 出現し得るアーティア武器を全て所持している仮定での検索も可能です。「スキル選択」画面の「特殊検索オプション」内にあります。
    - 初期設定では全グループスキル、全シリーズスキルが対象になっています。
    - 出現しないスキルがあった場合、MHWilds_SKILL.csvの「アーティア対応」列の「1」を消すと、対象から外れます。
- 限界突破強化関連
  - 限界突破強化の有無を検索時に指定できます。
  - 防御力の強化値はMHWilds_DEF_UPGRADE.csvで定義しています。
    - 値は仮置きで、正式版提供時に修正予定です。正確な値が欲しい方は実機に合わせて調整してください。
  - レア5,6のスロット強化は、以下の4種を用意しています。conf/LogicConfig.csvの「(仮)限界突破スロットの挙動」に対応する番号をいれることで選択できます(初期設定0)。
    - 0:レア5は全スロットを+1(計+3)、レア6はスロット1,2を+1(計+2)。元々Lv3のスロットは変化しない
      - (例)ミツネアームβ,3-0-0→3-1-0
    - 1:レア5は全スロットを+1(計+3)、レア6はスロット1,2を+1(計+2)。元々Lv3のスロットの拡張分は、次のスロットの拡張に繰り上げされる
      - (例)ミツネアームβ,3-0-0→3-2-0
    - 2:元々Lv3のスロットを除いて、左から順にレア5は3つ、レア6は2つ拡張(各スロット+1までで終了)
      - (例)ミツネアームβ,3-0-0→3-1-1
    - 3:元々Lv3のスロットを除いて、左から順にレア5は3つ、レア6は2つ拡張(余ったらループ)
      - 現状の装備では2と同じ
    
## 概要

Windows上で動くモンハンワイルズスキルシミュレータです

## 更新・対応状況

本体の最終更新：2025/12/12 12/11のショーケースの内容に合わせて限界突破強化のスロット変化の仕様を変更、護石登録時のスロット操作UIにクリアボタンを追加

CSVの最終更新：2025/12/12 シルドコートが足装備になっていたので修正

今までの更新履歴は[ReleaseNote.md](./ReleaseNote.md)を参照

## 特徴

- 各種データをCSV形式(一部json形式)で保持しています
  - 発売直後やイベクエ追加時など、装備等の追加を、シミュの更新を待たずに自身で行えます
- ランダム入手の護石を登録して検索に使えます
- 検索結果が0件だった場合に、どんな護石(光るお守り)があれば実現可能か検索できます
- 出現しうる護石(光るお守り)をすべて含めた状態で検索・追加スキル検索ができます
- 「最近検索に利用したスキル」からスキルを選択できます
- 「よく使う検索条件」を保存して、必要な時に呼び出すことができます
- ソースを公開しています
  - ご自由に自分好みにカスタマイズしてください

## 注意点

- 64bitマシンのWindowsでしか動きません
- .Netのインストールが必要です(無ければ起動時に案内されるはず)
  - x64のデスクトップランタイムで動くはず
  - 持ってない場合は、Visual C++ 再頒布可能パッケージも必要
- ファイルにデータを書き出す(マイセットとかの保存用)ので、ウィルス対策ソフトが文句言ってくる場合があります
- スキルによる防御や耐性上昇は計算していません
- 装飾品の組み合わせは1通りしか検索しないため、他の組み合わせが可能な場合もあります
  - 例：研磨・達人珠【３】1つと攻撃珠【１】1つ→研磨・攻撃珠【３】1つと達人珠【１】1つ
- 検索アルゴリズムの仕様上、1度に1000件や2000件検索するのはとても重くなります
  - 大量に検索するより、追加スキル検索や除外・固定機能をうまく使って絞り込む方がいいです
